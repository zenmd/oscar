2.6.2     Navigator for the program page.

 

When in the Program page, the Navigator lists the various pages (screens) associated with an individual program including the summary, queue, staff, clients, incidents and service restrictions pages. Some pages or links may not be accessible to all users, the access to these pages and links may be controlled by the system access security. See chart for further information.

 

 

 

 

 

Figure 6: Program Navigator

 

 

Label

Label

Action

A

Bronson Program (206888)

Display of program name and program number being worked on.

B

General

By selecting this option users will access the program’s general information page. The program’s general information page provides the tools necessary to create and maintain the basic information related to the definition of a program. Some of the information on this page includes: name and description of program, capacity information, and sector and restriction information.

C

Queue

By selecting this option users will access the program’s queue page. The program’s queue page lists clients that are waiting in the “queue” so that they may be assessed for possible admission into the program. Clients are placed in the queue by internal and external referrals.

D

Staff

By selecting this option users will access the program’s staff page. The program’s staff page lists the staff that has access to individual program.

E

Client

By selecting this option users will access the program’s client page. The program’s client page lists all the clients that are currently admitted on the program. The page also allow user to do batch discharges for multiple clients at a time that are admitted on the program.

F

Incidents

By selecting this option users will access the program’s incident pages. The program’s incident pages provide the tools necessary to create and maintain incidents using the incident form on these pages.

G

Service Restrictions

By selecting this option users will access the program’s service restriction pages. The program’s service restriction pages provide the tools necessary to create and maintain some additional variables regarding service restrictions such as the maximum length of restrictions and the default service restriction length.
